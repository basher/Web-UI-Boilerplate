# Forms

## Accessibility considerations
- All form fields need a `<label>`, which is explicitly associated with the relevant field with `for | id` attribute pairs.
- Additionally, any "grouped" form fields (e.g. `<input type="radio">`) should be enclosed in a `<fieldset>` with an appropriate `<legend>` to provide additional context for screen readers.
- Invalid fields should be identified with `aria-invalid="true"` and `aria-describedby` which references the `id` of appropriate error text.

## Native HTML5 browser validation
- Using `:valid`, `:invalid` and `:required` CSS pseudo classes.
- Adding a `title` attribute to a form field can improve the default browser validation error message.
