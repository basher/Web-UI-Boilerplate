## Accessibility and responsive behaviour considerations
- Rather than stacking tabs vertically on narrow viewports, a simple horizontal overflow scrolling solution is used instead to make tabs responsive.
- Tabs are fully keyboard accessible:
    - Use <kbd>tab</kbd> key to focus on currently selected tab, and traverse the content inside the tab.
    - Use <kbd>shift</kbd> + <kbd>tab</kbd> key to move backwards.
    - Use <kbd>right</kbd> arrow key to focus on next tab.
    - Use <kbd>left</kbd> arrow key to focus on previous tab.
    - Use <kbd>home</kbd> key to focus on first tab.
    - Use <kbd>end</kbd> key to focus on last tab.
- See the [ARIA APG tabs pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/examples/tabs-automatic/) and [Inclusive Components tabs](https://inclusive-components.design/tabbed-interfaces/) examples.

## Required and optional HTML  or `data-` attributes

Attribute | Behaviour
--- | ---
`data-tablist` | Required. Add to `<div>` which contains links that will be transformed into tab "triggers".
`data-tab` | Required. Add to tab "triggers".
`data-tabpanel` | Required. Add to content that will be shown/hidden.
`data-vertical` | Optional. Add to parent `<webui-tabs>` to create vertical tabs.
